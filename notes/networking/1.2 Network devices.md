## Networking Devices

## Router (Layer 3)
- Routes data from one IP to another IP within a network or different part of the world
- Connects diverse network types such as LAN, WAN
- IP address, routing table, hop
- **Security angle**: routers can enforce ACLs (Access Control Lists) to permit/deny traffic by IP — often the first point of network segmentation; misconfigured routers can expose internal networks

## Switch (Layer 2)
- Forwards network traffic based on MAC address
- PoE (Power over Ethernet) — can power devices like APs/cameras over the same cable
- MAC address table, ports, PoE
- **Security angle**: vulnerable to MAC flooding (overwhelming the MAC table to force the switch into hub-like broadcast mode) and VLAN hopping attacks

## Firewall
- Traditional firewalls filter traffic by TCP or UDP port number
- NGFW (Next-Generation Firewall) goes beyond port-based filtering to identify specific applications traversing the network, allowing administrators to allow or block them accordingly
- Has VPN capability that enables encrypted tunnels between remote sites
- Often sits at the ingress/egress points of a network; can perform routing functions, including Network Address Translation (NAT) and dynamic routing
- NGFW, NAT, ingress/egress, application-aware filtering
- **Security angle**: the core perimeter defense device — most security architectures start with "what does the firewall allow/deny"; firewall misconfiguration is a very common cause of real breaches

## IDS and IPS
- Systems monitor network traffic for known attacks called intrusions
- Intrusion Detection System (IDS) alerts administrators to threats (passive — detects and logs, doesn't block)
- Intrusion Prevention System (IPS) actively blocks them before they enter the network (active — sits inline, can stop traffic)
- Signature-based detection, anomaly-based detection, inline vs out-of-band
- **Security angle**: this is core SOC/blue team territory — IDS/IPS alerts often feed directly into a SIEM for analysis; understanding false positives vs true positives here is a daily SOC analyst task

## Load Balancer
- Networking device used to distribute incoming traffic across multiple physical servers, ensuring high availability and preventing downtime — makes backend infrastructure appear to the end-user as a single service
- If a server fails due to hardware or software issues, the load balancer detects the outage, removes that server from rotation, and continues providing service using remaining healthy servers
- Can perform TCP offloading to speed up communication and manage SSL offloading, which handles encryption/decryption tasks to reduce processing burden on individual servers
- By caching data on the load balancer, requests can be answered immediately without querying backend servers
- Uses QoS to prioritize certain types of traffic, or application-centric load balancing to route specific requests (like certain web pages) to designated servers
- High availability, SSL offloading, failover
- **Security angle**: SSL offloading means the load balancer decrypts traffic — meaning traffic between the load balancer and backend servers may be unencrypted internally, which matters for internal network security assumptions; also a common target for DDoS mitigation configuration

## Proxy
- Acts as an intermediary device positioned between a user and an external network (like the Internet) — instead of a user communicating directly with a remote server, the proxy takes the request, performs it on the user's behalf, verifies the response, and provides the answer back to the user
- Because the proxy sits in the middle of the conversation, it can cache content, allowing future requests for the same data to be served immediately without needing to access the Internet again
- Useful for URL filtering, content scanning, access control
- Can be **explicit** (requiring manual configuration in the user's applications/OS) or **transparent** (functions invisibly, no user-side config needed)
- Forward proxy, caching, URL filtering
- **Security angle**: forward proxies are used to enforce corporate content/access policy and log outbound traffic; attackers may also abuse open/misconfigured proxies to hide their origin or bypass network restrictions

## NAS and SAN
- **Network Attached Storage (NAS)**: provides file-level access — to read or modify a file, your system must pull the entire file across the network into local memory, then write the entire modified file back to the NAS
- **Storage Area Network (SAN)**: provides block-level access, behaving like a local storage drive — instead of transferring an entire file, you only modify the specific blocks of data that changed
- File-level vs block-level access
- **Security angle**: both are common targets for ransomware (encrypting shared/mounted storage can affect every connected user); access controls and permissions on these are critical since a single misconfigured share can expose large amounts of data

## AP (Access Point)
- A purpose-built networking device that enables wireless communication between devices and the wired network
- Acts as an OSI Layer 2 (Data Link) device by bridging communication between the 802.11 wireless network and the 802.3 wired Ethernet network
- In larger environments, organizations use multiple access points for seamless coverage; since managing these individually is difficult, administrators use a Wireless LAN Controller (WLC) for centralized deployment, security monitoring, and configuration updates
- 802.11, WLC, bridging
- **Security angle**: rogue APs (unauthorized access points) are a common attack vector for intercepting wireless traffic; weak wireless encryption (e.g., WEP, weak WPA configs) at the AP level is a frequent pentest finding
