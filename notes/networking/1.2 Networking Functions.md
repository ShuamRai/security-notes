## Networking Functions

## Content Delivery Network (CDN)
- A geographically distributed network of servers designed to cache and deliver data efficiently to end-users, significantly reducing latency compared to accessing a centralized server
- Users get data from a local/nearby server instead of one central origin server
- Edge server, caching, latency reduction
- **Security angle**: CDNs (e.g., Cloudflare) also absorb DDoS attacks by distributing traffic load across many servers and filtering malicious traffic at the edge, before it reaches the origin server

## Virtual Private Network (VPN)
- A method for connecting to a remote network securely by encrypting data across public mediums (like the internet)
- Often utilizes a VPN concentrator or a next-generation firewall as the central connection point
- Encryption, tunneling, VPN concentrator
- **Security angle**: VPNs protect data confidentiality on untrusted networks (e.g., public Wi-Fi), but attackers can also target VPN endpoints/credentials as an entry point — stolen VPN credentials are a common initial access vector in real breaches

## Quality of Service (QoS)
- Also known as traffic or packet shaping, this allows administrators to prioritize specific applications (like real-time audio/video) over others based on bandwidth and data rates
- Prevents congestion from degrading time-sensitive traffic (e.g., VoIP, video calls) by giving it priority over less urgent traffic (e.g., downloads)
- Traffic prioritization, bandwidth management
- **Security angle**: operates at Layer 3/4 (identifying traffic by IP/port/protocol); relevant during incident response to understand why certain traffic was throttled or prioritized on a network

## Time to Live (TTL)
- A protocol mechanism used to ensure tasks or data don't persist indefinitely
- In IP routing, it defines the maximum number of hops a packet can take before being discarded, which is vital for preventing routing loops
- In DNS, it specifies the duration (in seconds) that a resolution should be cached locally before expiring
- Hop count, DNS caching, routing loop prevention
- **Security angle**: attackers use TTL analysis (`ping`, `traceroute`) for OS fingerprinting and network mapping/reconnaissance; short DNS TTLs are also abused in "fast flux" attacks where malicious domains rapidly change IPs to evade blocklists
